# 八字命理解读 · 卦师模式

## 与普通模式的区别

| | 普通模式（bazi） | 卦师模式（bazi-master） |
|---|---|---|
| 输出风格 | 结论式，客户友好 | 判断链式，展示完整推理 |
| 适用对象 | 客户 | 卦师/自己调试 |
| 性格验证 | 有，逐句确认 | **跳过**，直接输出判断链 |
| 内部标签 | 禁止"第几句" | 同样禁止 |

**相同部分**：排盘核心、词汇表、断语库完全复用。

---

## ⚠️ vocabulary.md 的正确定位（重要）

**vocabulary.md = 翻译字典/桥梁，不是匹配规则。**

断语 ↔ vocabulary ↔ 排盘原始数据

它的作用是告诉你**"断语里的术语在数据里长什么样"**，而不是"看到数据去套哪个断语"。

| 场景 | 正确用法 |
|------|---------|
| 看到断语"偏印旺而无制" | 去 vocabulary 查：旺而无制 = isWang=1 && isShouZhi=0 |
| 排盘数据 isWang=1, isShouZhi=1 | 用八字理论自己推导这叫"旺而受制"，结合断语库理解含义 |

**❌ 错误用法**：
看到 isWang=1 → 去 vocabulary 套"旺而无制"规则 → 硬凑断语

**✅ 正确用法**：
看到 isWang=1, isShouZhi=1 → 这是"旺而受制"状态 → 结合八字理论 + 断语库 → 自己推导结论

**简单说**：vocabulary 让你看懂数据长什么样，不是操作手册。

## 交互流程

### 第一步：接收输入
接收用户出生时间（年月日时）+ 性别。
- 先确认日期类型：公历还是农历？
- 例："1990年8月27日 12:00 男"

### 第二步：展示命盘（完整数据）
排盘完成后，先将完整命盘数据发给客户，包含：
- 四柱（年、月、日、时）+ 纳音
- 藏干 + 主要神煞
- 身强身弱结论
- 十神旺衰喜忌一览

### 第三步：话题确认
确认客户想了解的话题：

```
想继续了解哪个？
💍 感情/婚恋
💰 事业/财运
👶 子女
📚 学业/考试
🏥 健康/身体
👨‍👩‍👧 父母/家人
🔮 其他（指定话题）
```

### 第四步：断事输出（判断链模式）

⚠️ **卦师模式核心**：输出时必须展示完整推理判断链，每一步都要写出来。

**判断链格式：**

```
【话题】学业

1. 条件判断：
   - 印星 isWang=1, isShouZhi=1 → 印星旺而受制（能量强但不流通）
   - 官星 isWang=1, isShouZhi=1 → 官星旺而受制
   - 文昌贵人在月柱

2. 特征推导（用八字理论结合断语库）：
   - 印星旺而受制 → 印星能量发挥受限，学业助力不稳定
   - 官星旺而受制 → 压力能量发挥受限，考学压力难以转化
   - 身强比肩旺 → 考试竞争中共者多，容易被人影响

3. 综合结论：
   学业运中等，有文昌加持本有学历潜质，但印星受制，学习助力不稳定。
   身强比肩旺，竞争环境中心态容易波动。
   建议补印星之力（文科、书房文昌位），规避比肩忌神环境。
```

**判断链规则：**
- 必须包含"条件判断 → 特征推导 → 综合结论"三段式
- 每一步都要写，不跳过
- 条件判断写原始数据（isWang/isShouZhi/exists 等数值）
- 特征推导用八字理论结合断语库，不硬套 vocabulary
- 综合结论从判断自然推导，不是凭空加的

---

### 第五步：输出前核查（必须执行）

⚠️ **所有断语输出前必须过核查，防止男女命混淆等低级错误**

用 DeepSeek 做审查员，逐条核对：

```
你是八字知识审查员。请核对以下断语是否符合知识清单：

1. 核对男女命规则是否用对（男命夫妻星=正偏财，女命夫妻星=正官七杀）
2. 核对六亲对应的十神是否正确
3. 核对墓库是否正确
4. 核对长生状态是否正确

断语：[用户要输出的内容]
性别：[男命/女命]

如果有问题，指出错误并给出修正建议。
如果没问题，回复"核对通过"。
```

**流程：**
1. 生成断语草稿
2. 发给 DeepSeek 审查
3. DeepSeek 回复"核对通过" → 批准输出
4. DeepSeek 指出错误 → 修正后重新审查

**知识清单文件：** `../bazi/references/knowledge_checklist.md`

## 执行排盘

复用 `bazi` skill 的脚本：

```bash
cd /root/.openclaw/workspace/skills/bazi/scripts
/root/.nvm/versions/node/v22.22.1/bin/node bazi.js 1990-08-27 12:00 男
```

参数格式：
- 日期：1990-01-01 或 1990年1月1日
- 时间：12:00 或 12点
- 性别：男 或 女
- 农历：用 `--lunar` 参数
